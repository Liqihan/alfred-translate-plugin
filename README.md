# Alfred LLM Translator

Alfred LLM Translator is an Alfred workflow that translates `tr <text>` through an OpenAI Chat Completions compatible API and pastes the result into the active macOS application when you press Enter.

## Features

- Keyword workflow: `tr <text>`.
- Node.js implementation with no third-party runtime dependencies.
- OpenAI Chat Completions compatible endpoint support.
- User-managed config at `~/.alfred-llm-translator.json` so secrets are never bundled in release artifacts.
- Automatic target language rule: Chinese input translates to English; other input translates to Simplified Chinese.
- Clear Alfred error items for empty input, missing or invalid config, API failures, timeouts, malformed responses, and empty model output.
- Directly installable `alfred-llm-translator.alfredworkflow` release artifact generated by GitHub Actions, not committed to the repository.

## Requirements

- macOS with Alfred installed.
- Node.js 18 or newer available on the PATH Alfred sees.
- An OpenAI Chat Completions compatible API key, base URL, and model name.

## Install From GitHub Releases

1. Download `alfred-llm-translator.alfredworkflow` from the latest GitHub Release.
2. Double-click the file on macOS.
3. Alfred opens an import prompt; confirm the import.
4. Create the config file described below.

Users should not need to clone this repository or run build commands when installing from a release. The `.alfredworkflow` binary artifact is intentionally produced by GitHub Actions and attached to releases instead of being committed to git.

## Configuration

Create `~/.alfred-llm-translator.json`:

```json
{
  "apiKey": "sk-...",
  "baseUrl": "https://api.openai.com/v1",
  "model": "gpt-4.1-mini",
  "temperature": 0.2,
  "timeoutMs": 15000,
  "targetRule": "auto_zh_en",
  "promptTemplate": "You are a professional translation engine. Translate the following text into {{targetLanguage}}. Preserve meaning, tone, formatting, punctuation, URLs, code, and product names. Return only the translation, no explanation.\n\nText:\n{{text}}"
}
```

Required fields:

- `apiKey`
- `baseUrl`
- `model`

Optional fields and defaults:

- `temperature`: `0.2`
- `timeoutMs`: `15000`
- `targetRule`: `auto_zh_en`
- `promptTemplate`: the professional translation prompt shown above

The workflow normalizes trailing slashes in `baseUrl` and calls:

```text
<baseUrl>/chat/completions
```

For example, `https://api.openai.com/v1/` becomes `https://api.openai.com/v1/chat/completions`.

## Usage

Open Alfred and type:

```text
tr hello world
```

Expected behavior:

1. Alfred shows the translated result.
2. The subtitle includes the target language and model name.
3. Press Enter to copy the translation to the clipboard and send Command+V to the active app.

If the config is missing or invalid, Alfred shows a fixable error item. Press Enter on config-related errors to create or open `~/.alfred-llm-translator.json`.

## Translation Rule

Version 1 supports one rule: `auto_zh_en`.

- Input containing Chinese characters translates to English.
- All other input translates to Simplified Chinese.

The prompt template supports these placeholders:

- `{{targetLanguage}}`
- `{{text}}`

## Troubleshooting

### Alfred says the config is missing

Press Enter on the error result or manually create `~/.alfred-llm-translator.json`. The workflow will create an example config with file mode `0600` when possible.

### Alfred says required fields are missing

Ensure `apiKey`, `baseUrl`, and `model` are non-empty strings.

### Requests time out

Increase `timeoutMs`, verify your network, and confirm the provider's `baseUrl` is reachable from macOS.

### API returns HTTP errors

Check your API key, model name, provider account status, and whether the provider supports the OpenAI Chat Completions request shape.

### Nothing pastes

The paste action uses `pbcopy` and AppleScript (`System Events`) to send Command+V. macOS may ask for Accessibility permission for Alfred or the script runner.

## Development

Run tests:

```sh
npm test
```

Build the release artifact:

```sh
npm run build
```

The build command creates `alfred-llm-translator.alfredworkflow` locally. The artifact includes workflow files, scripts, package metadata, README, and the design spec. It excludes local config files and API keys. The generated `.alfredworkflow` is ignored by git; GitHub Actions builds and uploads it for pull requests and attaches it to tagged releases.

## Release Checklist

1. Open or merge a pull request and confirm the GitHub Actions workflow passes.
2. Create and push a version tag such as `v1.0.0`.
3. GitHub Actions builds `alfred-llm-translator.alfredworkflow` and attaches it to the tagged GitHub Release.
4. Download the release artifact and verify a fresh Alfred import can run `tr hello` with a valid config.
